# どんぶり勘定選手権(Webアプリ版)

10人程度で遊ぶ、フェルミ推定パーティーゲーム(チーム数は進行役がロビー画面で2〜8チームに変更可能)。
ルールの詳細・設計意図は [`../原案/フェルミ推定ミドルゲーム.md`](../原案/フェルミ推定ミドルゲーム.md) を参照。

## 構成
- 進行役(1人)が **ノートPCなど1台** でサーバーを起動し、`host.html` をテレビ/プロジェクターなど共有画面に映す
- 参加者は各自の **スマホ** から `team.html` にアクセスして名前を入力すると、自動的にいずれかのチームに割り振られる(人数が少ないチームに優先的に割り振り)
- 進行・回答・タイマー・集計はすべてリアルタイムに同期される(Socket.IO)
- ローカル起動時は同じWi-Fi内限定。無料サーバーにデプロイすれば、参加者は同じWi-Fiでなくてもインターネット経由で参加できる(手順は後述)

## 起動方法
```
cd webapp
npm install   # 初回のみ
npm start
```

起動するとターミナルに以下が表示される。

- 進行役用URL: `http://localhost:3000/host.html`(サーバーを起動したPC自身で開く)
- 参加者用URL + QRコード: `http://<進行役PCのIPアドレス>:3000/team.html`(参加者はQRコードを読み取ってアクセスするのが簡単)

進行役PCと参加者のスマホが**同じWi-Fiに接続されている**必要がある。

## 進行の流れ
1. 参加者が各自スマホから参加(名前入力のみ)。ロビー画面で進行役がチーム数(2〜8)・チーム名・出題するお題を編集できる
2. 進行役画面で「ゲーム開始」を押す(開始後はチーム数・チーム名・お題選択を変更できない)
3. お題が表示され、5分間の議論タイマーがスタート。各チームは代表者がスマホから数値を送信(締切まで何度でも再送信可)
4. 全チーム提出、または時間切れで自動的に締め切り
5. 進行役が「せーので開示する」を押すと、全チームの回答が同時に公開され、平均値に最も近いチームに1ポイント
6. 「次の問題へ」で次のラウンドへ。選択したお題が全て終わると最終結果画面が表示される
7. 「もう一度遊ぶ」でスコア・参加者をリセットして最初から遊べる(参加者はスマホで再度名前入力が必要。チーム数・チーム名・お題バンクの設定は引き継がれる)

## 無料サーバー(Render.com)へのデプロイ
同じWi-Fiでなくても参加できるようにする場合は、Render.comの無料プランにデプロイする。
GitHubアカウントの作成・連携はブラウザでの本人操作が必要なため、以下の手順を進めてほしい。

1. https://github.com でアカウントを作成(未取得の場合)
2. GitHub上で新しいリポジトリを作成(例: `fermi-middle-game`。Public/Privateどちらでも可。READMEなどの初期ファイルは追加しない)
3. 作成したリポジトリのURLをこのプロジェクトのリモートに設定してpush
   ```
   cd webapp
   git remote add origin https://github.com/<あなたのユーザー名>/<リポジトリ名>.git
   git branch -M main
   git push -u origin main
   ```
   (pushの際にGitHubのログインを求められたら、ブラウザでの認証を完了させる)
4. https://render.com にアクセスし、「Continue with GitHub」でサインアップ
5. ダッシュボードで「New +」→「Web Service」→ 先ほどのリポジトリを選択
6. このリポジトリには `render.yaml` を含めてあるので、ビルド/起動コマンドは自動設定される(念のため Build Command: `npm install` / Start Command: `npm start` / Plan: Free になっているか確認)
7. 「Deploy」を押してビルド完了を待つ(数分)
8. 発行された `https://<サービス名>.onrender.com` が本番URL。進行役は `/host.html`、参加者には `/team.html` を共有する

### 無料プランの制約
- 15分間アクセスがないと自動でスリープし、次のアクセス時に**起動まで30〜50秒程度**かかる
- 遊ぶ前に進行役が数分早めに `host.html` を開いてサーバーを起こしておくとスムーズ
- コードを更新した場合は `git push` するとRenderが自動で再デプロイする

## お題の編集
起動時の初期お題は `data/questions.json` に入っており、これを直接編集しても差し替えられる。
それに加えて、進行役はロビー画面から**アプリ上で直接**お題の追加・削除・出題する問題の選択ができる(ゲーム開始前のみ操作可能)。
- 追加したお題やチェックボックスでの選択状態は、サーバーを起動している間はメモリ上に保持される(`data/questions.json` 自体は書き換えない。サーバー再起動やRenderの再デプロイ・スリープ復帰でリセットされる点に注意)
- 出題基準(公表されている実数値を避ける、など)は [`../原案/フェルミ推定ミドルゲーム.md`](../原案/フェルミ推定ミドルゲーム.md) の「お題選定基準」を参照

## 既知の制約(現状のシンプル実装によるもの)
- 参加者がページを再読み込みすると再参加(名前入力)が必要になる(接続が切れるとサーバー側の登録も削除されるため)
- チームの回答は「そのチームの誰かが最後に送信した値」が採用される(チーム内合意の確認機能はなし。声を掛け合って1人が代表送信する運用を想定)
- 全チーム未提出のまま強制締め切りした場合、未提出チームの回答は便宜上0として扱われる
- ロビーで追加・選択したお題はサーバーのメモリ上のみに保持され、`data/questions.json` には保存されない(サーバー再起動で初期6問に戻る)
