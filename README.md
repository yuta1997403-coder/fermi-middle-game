# フェルミ推定ミドルゲーム(Webアプリ版)

10人・3チームで遊ぶ、フェルミ推定パーティーゲーム。
ルールの詳細・設計意図は [`../原案/フェルミ推定ミドルゲーム.md`](../原案/フェルミ推定ミドルゲーム.md) を参照。

## 構成
- 進行役(1人)が **ノートPCなど1台** でサーバーを起動し、`host.html` をテレビ/プロジェクターなど共有画面に映す
- 参加者は各自の **スマホ** から `team.html` にアクセスして名前を入力すると、自動的に3チームのいずれかに割り振られる(人数が少ないチームに優先的に割り振り)
- 進行・回答・タイマー・集計はすべてリアルタイムに同期される(Socket.IO)
- 身内利用専用。同じWi-Fiに繋がっていれば追加設定なしで遊べる(インターネット公開はしていない)

## 起動方法
```
cd webapp
npm install   # 初回のみ
npm start
```

起動するとターミナルに以下が表示される。

- 進行役用URL: `http://localhost:3000/host.html`(サーバーを起動したPC自身で開く)
- 参加者用URL + QRコード: `http://<進行役PCのIPアドレス>:3000/team.html`(参加者はQRコードを読み取ってアクセスするのが簡単)

進行役PCと参加者のスマホが**同じWi-Fiに接続されている**必要がある。

## 進行の流れ
1. 参加者が各自スマホから参加(名前入力のみ)
2. 進行役画面で「ゲーム開始」を押す
3. お題が表示され、5分間の議論タイマーがスタート。各チームは代表者がスマホから数値を送信(締切まで何度でも再送信可)
4. 全チーム提出、または時間切れで自動的に締め切り
5. 進行役が「せーので開示する」を押すと、3チームの回答が同時に公開され、中央値(=平均に一番近い)を出したチームに1ポイント
6. 「次の問題へ」で次のラウンドへ。全6問終了後、最終結果画面が表示される
7. 「もう一度遊ぶ」でスコア・参加者をリセットして最初から遊べる(参加者はスマホで再度名前入力が必要)

## お題の編集
`data/questions.json` を編集すると出題内容を差し替えられる。
出題基準(公表されている実数値を避ける、など)は [`../原案/フェルミ推定ミドルゲーム.md`](../原案/フェルミ推定ミドルゲーム.md) の「お題選定基準」を参照。

## 既知の制約(現状のシンプル実装によるもの)
- 参加者がページを再読み込みすると再参加(名前入力)が必要になる(接続が切れるとサーバー側の登録も削除されるため)
- チームの回答は「そのチームの誰かが最後に送信した値」が採用される(チーム内合意の確認機能はなし。声を掛け合って1人が代表送信する運用を想定)
- 全チーム未提出のまま強制締め切りした場合、未提出チームの回答は便宜上0として扱われる
